# راهنمای کامل استفاده از v2ray-finder

این راهنما برای کسانی نوشته شده که اطلاعات فنی زیادی ندارند. قدم به قدم همه چیز توضیح داده شده.

---

## 🌐 نسخه تحت وب (بدون نصب)

اگر نمی‌خواهید چیزی نصب کنید، نسخه آنلاین پکیج در آدرس زیر در دسترس است:

**👉 [rkarimabadi.github.io/v2ray-finder-dotnet](https://rkarimabadi.github.io/v2ray-finder-dotnet)**

این نسخه از طریق مرورگر لینک‌های V2Ray را جمع‌آوری و نمایش می‌دهد — بدون هیچ نصبی. با این حال، به دلیل محدودیت‌های محیط GitHub Pages، امکان **Health Check** (بررسی سلامت سرورها) در این نسخه وجود ندارد. اگر به این قابلیت نیاز دارید، از نسخه Python استفاده کنید.

---

## v2ray-finder چیست؟

این ابزار به طور خودکار لینک‌های V2Ray رایگان را از منابع مختلف اینترنت جمع‌آوری، تکراری‌ها را حذف و کیفیت هر سرور را بررسی می‌کند. خروجی یک لیست تمیز از لینک‌های آماده استفاده است — از نوع `vmess://`، `vless://`، `trojan://`، `ss://` و `ssr://`.

---

## قدم ۱ — نصب Python

قبل از هر چیز، Python روی کامپیوتر شما باید نصب باشد (نسخه 3.8 به بالا).

- برای بررسی: ترمینال (یا Command Prompt در ویندوز) را باز کنید و تایپ کنید: `python --version`
- اگر نصب نیست، از [python.org](https://www.python.org/downloads/) دانلود کنید.

---

## قدم ۲ — نصب پکیج

در ترمینال، یکی از دستورات زیر را اجرا کنید:

```bash
# نصب ساده (پایه‌ای‌ترین حالت)
pip install v2ray-finder

# نصب کامل (توصیه می‌شود — همه امکانات)
pip install "v2ray-finder[all]"
```

> **نکته:** نصب کامل شامل رابط گرافیکی، واسط ترمینال زیبا، و سرعت بیشتر می‌شود.

---

## قدم ۳ — تهیه GitHub Token (اختیاری اما مهم)

برای این مرحله به یک **اکانت GitHub** نیاز دارید. اگر اکانت ندارید، رایگان از [github.com/signup](https://github.com/signup) بسازید.

بدون token فقط ۶۰ درخواست در ساعت می‌توانید بزنید. با token این عدد به ۵۰۰۰ می‌رسد.

1. وارد اکانت GitHub خود شوید
2. به [github.com/settings/tokens](https://github.com/settings/tokens) بروید
3. روی «Generate new token» کلیک کنید
4. تیک `public_repo` را بزنید و token را بسازید
5. Token را کپی کنید (فقط یک بار نشان داده می‌شود — جای امنی ذخیره کنید)
6. در ترمینال اجرا کنید:

```bash
# لینوکس / مک
export GITHUB_TOKEN="ghp_توکن_شما_اینجا"

# ویندوز (Command Prompt)
set GITHUB_TOKEN=ghp_توکن_شما_اینجا
```

## قدم ۴ — استفاده از رابط گرافیکی (GUI)

ساده‌ترین روش برای کسانی که با ترمینال راحت نیستند:

```bash
v2ray-finder-gui
```

پنجره‌ای باز می‌شود با این امکانات:

| بخش | توضیح |
|-----|-------|
| دکمه Start | شروع جستجو و بررسی سرورها |
| دکمه Stop | توقف در هر لحظه |
| نوار Progress | نشان‌دهنده پیشرفت کار |
| جدول نتایج | ۷ ستون: شماره، پروتکل، امتیاز، درجه، تأخیر، منبع، لینک |
| نوار آمار | تعداد پیداشده / منحصربه‌فرد / سالم / امتیازدارشده |
| بخش Failed Sources | منابعی که خطا داشتند |

---

## قدم ۵ — استفاده از ترمینال (CLI)

اگر با ترمینال کار می‌کنید، دستورات زیر مفید است:

```bash
# اجرای ساده — لیست سرورها را نشان می‌دهد
v2ray-finder

# ذخیره در فایل
v2ray-finder -o servers.txt

# جستجوی گسترده‌تر در GitHub + محدود کردن به ۲۰۰ سرور
v2ray-finder -s -l 200 -o servers.txt

# فقط نمایش آمار
v2ray-finder --stats-only

# فقط سرورهای سالم با کیفیت بالا
v2ray-finder -c --min-quality 60 -o healthy_servers.txt
```

**ترمینال زیباتر (Rich CLI):**

```bash
v2ray-finder-rich
```

---

## قدم ۶ — بررسی سلامت سرورها (Health Check)

این پکیج سرورها را در ۳ لایه چک می‌کند:

- **لایه ۱** — اتصال TCP: آیا سرور اصلاً در دسترس است؟
- **لایه ۲** — پروب HTTP: آیا پاسخ می‌دهد؟
- **لایه ۳** — تست واقعی از طریق xray + Google: آیا واقعاً اینترنت را باز می‌کند؟

برای فعال‌کردن health check از CLI:

```bash
v2ray-finder -c --min-quality 60 -o healthy_servers.txt
```

---

## سیستم امتیازدهی

هر سرور یک امتیاز از ۰ تا ۱۰۰ می‌گیرد و یک درجه A تا F:

- **A** = عالی (سرعت بالا، تأخیر کم، قابل اعتماد)
- **F** = ضعیف یا غیرقابل استفاده

امتیاز بر اساس ۷ معیار محاسبه می‌شود: تأخیر، دسترسی‌پذیری، پروتکل، اعتبار منبع، تازگی، منحصربه‌فردی، و تست Google 204.

---

## خروجی را کجا و چطور استفاده کنم؟

محتوای فایل `servers.txt` را کامل کپی کنید، سپس در کلاینت V2Ray خود گزینه **Import from Clipboard** را بزنید — همه لینک‌ها به یکباره وارد می‌شوند. این روش در همه کلاینت‌های رایج پشتیبانی می‌شود:

- **v2rayNG** (اندروید)
- **v2rayN** (ویندوز)
- **Nekoray** (لینوکس)

پس از ایمپورت، یک بار **TCPing Test** یا **Real Delay Test** بگیرید تا سرعت واقعی هر سرور سنجیده شود. سپس لیست را بر اساس کمترین ping سورت کنید و از بهترین سرور بالای لیست استفاده کنید.
